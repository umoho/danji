# 放松求解器容差到 1e-6

## 改动

`src/circuit/solver.rs`:
```
- const TOL: f64 = 1e-9;
+ const TOL: f64 = 1e-6;
```

## 理由

实时场景下 1e-9 过严。部分 12AX7 级联场景下，Newton 迭代需要
40-50 次才能收敛到 1e-9，在接近收敛区（delta 1e-7~1e-5）形成极限环。

1e-6 对应 0.3µV 精度（300V B+ 下），远低于管子热噪声和 DAC 量化噪声。

## 性能影响

Bench (single 12AX7, 44.1kHz, release build):

| TOL | 实时倍率 | 每样本 |
|-----|---------|--------|
| 1e-9 | 13.85x | 1.64µs |
| 1e-6 | 16.04x | 1.41µs |

**+16%**，无副作用。

## 验证

所有 18 个单元测试 + 4 个示例（pentode_stage, full_chain, push_pull, output_transformer）
均在 TOL=1e-6 下通过，输出值完全一致。
