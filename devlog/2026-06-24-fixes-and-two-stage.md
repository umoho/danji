## 2026-06-24

### 关键 Bug 修复

实现过程中发现并修复了以下关键问题：

#### 1. 后向欧拉电容 MNA 符号错误

**症状**: 电容无法正确充放电，耦合电容的直流偏置不正确，导致双级电路瞬态求解发散。

**根因**: `stamp_capacitor` 中历史电流源符号反转。

```
// 错误（原始代码）:
i[a] += gc * (v_b - v_a);  // 历史电流进入节点 a

// 正确（修复后）:
i[a] += gc * (v_a - v_b);  // 历史电流进入节点 a
```

根据 MNA 理论，电容的伴随模型为电导 `Gc = C/h` 并联历史电流源 `I_hist = Gc * V_cap[n-1]`。对于节点 a（电容正极），历史电流源应注入 `+Gc * (Va_prev - Vb_prev)`，而非 `+Gc * (Vb_prev - Va_prev)`。

**影响**: 修复后单管增益从 72x 变为 62x（更接近理论值 58x）。

#### 2. 矩阵条件数过大

**症状**: 双管 DC 求解发散，即使使用源步进法也失败。

**根因**: 电压源电导使用 1e12，与电阻电导 1e-5 相差 17 个数量级，导致矩阵严重病态。

**修复**: 
- 引入 `VSRC_G = 1e6` 替代硬编码的 1e12
- 降低电压源电导后，双管 DC 求解器收敛（20 步源步进）

#### 3. 数值微分精度不足

**症状**: 在小电压区域，雅可比矩阵元素 gp、gm 为零，导致牛顿迭代停滞。

**根因**: 固定微分步长 1e-6 在小电压下（如 Vpk = 1e-6）无法产生可测量的电流变化。

**修复**: 自适应步长 `eps = max(1e-6, |x| * 1e-4)`

### 双级预放实现

采用**独立级联**架构（非直接 MNA 耦合）：

```
Stage 1 (12AX7) → 耦合电容 HPF → Stage 2 (12AX7)
```

- 每级独立求解 MNA（避免耦合电容 BE 电导导致的数值问题）
- 级间耦合使用指数平滑 RC 滤波器: `Vout = Vcap + alpha * (Vin - Vcap)`
- 预热 3000 样本让电路达到静态工作点

**结果**:
- 总增益: 9022x (79.1 dB)
- 每级增益: ~62x
- 实时比: 14.95x (14.95:1)

### 当前局限

- **双级直接 MNA 耦合不收敛**: 22nF 耦合电容在 44.1kHz 采样率下，BE 等效电导 Gc = C/h = 0.00097 S (≈1031Ω)，远大于栅极电阻 1MΩ，导致直流通路错误。目前使用独立级联架构绕过此问题。
- **电容初值无关**: `cap.v_prev` 字段虽被更新但不被求解器使用。求解器使用自己的 `v_prev` 数组。

## 踩坑记录

### 坑 1: 电容初值设了但没人读

**现象**: 实现了 `compute_dc_bias` 计算直流偏置，然后在 `process_sample` 中更新 `cap.v_prev`。但瞬态求解器仍然从错误的状态开始。

**排查过程**:
1. 以为 `cap.v_prev` 是求解器使用的——打印确认求解器读的是 `self.v_prev`（求解器内部数组），不是 `cap.v_prev`
2. 阅读 `stamp_capacitor` 代码，确认其参数是 `&self.v_prev`
3. 又以为 `self.v_prev` 在每次 `solve()` 后被 `self.v_prev = self.v` 更新——但对瞬态求解的第一个样本，`v` 和 `v_prev` 都来自 DC 求解的结果，残差应为 0

**真相**: `cap.v_prev` 压根没人读。求解器只用自己的 `v_prev` 数组。设了也白设。

### 坑 2: 残差为零但牛顿不收敛

**现象**: 双级电路的瞬态求解，第一个样本收敛（3 次迭代），第二个样本进入极限环（delta 在 6e-3 和 7e-7 之间振荡）。

**排查过程**:
1. 添加迭代调试输出，发现 delta 在 3 次迭代后降到 7e-7，然后下一次牛顿步又回到 6e-3
2. 认为这是数值精度问题：残差在理论上为 0，但浮点误差产生 ~3e-7 的残差
3. 矩阵条件数 1e11（VSRC_G=1e6 和最小电导 1e-5），放大后 delta ≈ 1e5 * 3e-7 = 0.03
4. 改为相对容差 `TOL + max_v * 1e-9` → 但 0.03 >> 容差，不收敛
5. 尝试加大 TOL、增加迭代次数、移除步长钳位 → 都不行

**真相**: 根本原因不是容差，而是 Gc = C/h = 0.00097 S（≈1031Ω）在 44.1kHz 采样率下太大。这个 BE 等效电导把 V1 板和 V2 栅极强行拉到同一电位，电容永远充不上电。不是牛顿的问题，是 MNA 建模本身对 DC 耦合不适用。

### 坑 3: B+ 缓启 → 二极管特性导致发散

**现象**: 放弃 DC 求解后改用 B+ 缓启（transient solver，B+ 从 0 到 300V 步进）。单管工作，双管在 B+ ≈ 0.57V 时发散。

**排查过程**:
1. 打印每步 B+ 和节点电压: Vp1 和 Vg2 始终保持相等（耦合电容 Gc 强行拉平）
2. Vg2 跟随 Vp1 上升，但 Vk2 被旁路电容钳位在 0V
3. Vgk2 = Vg2 - Vk2 = Vp1 - 0 → 正偏压越来越大
4. 当 Vgk2 超过约 0.5V 时，12AX7 进入正栅压区，跨导剧变，牛顿振荡

**真相**: 耦合电容在 BE 模型下表现为 1031Ω 电阻，不是 DC 隔离。Vg2 不可能维持 0V 偏置。

### 坑 4: 两个独立 Simulator 间神秘干涉

**现象**: 同时创建 `sim1` 和 `sim2` 后，`sim1` 在第 5 个样本发散。单独创建 `sim1` 则工作正常。

**排查过程**:
1. 确认 `cfg.clone()` 创建了独立副本
2. 交换创建顺序: `sim2` 先、`sim1` 后 → 同样发散
3. 怀疑内存布局问题 → 但 Rust 保证安全
4. 用两个独立的 `single_triode_config()` 调用创建 cfg1 和 cfg2 → 正常！

**真相**: 用 `cfg.clone()` 时出了未知问题（可能是之前的 edits 在 SimConfig 或 Capacitor 的 Clone derive 上有残留 bug）。改用两个独立构造调用后解决。具体根因未完全查明，属于"修好了但不知道为什么"的坑。

### 坑 5: 最大迭代次数不足被误判为发散

**现象**: `MAX_ITER = 50` 时，`solve_dc` 对双管 300V 直接求解失败 → "Diverged"。但实际是牛顿在 200 次内能收敛。

**排查过程**:
1. 把 `MAX_ITER` 从 50 提到 200 → 还是发散
2. 说明不是迭代次数问题
3. 排查矩阵条件数 → 发现 1e12 vs 1e-5 差 17 个数量级
4. 降到 1e6 → 50 次迭代就收敛

**真相**: 表面是迭代次数不足，实则是矩阵病态导致牛顿步长方向错误，给再多迭代也收敛不了。

### 坑 6: 单管增益突变（72x → 498x → 62x）

**现象**: 
- 原始代码（错误电容符号）: 72x
- 修正电容符号后: 498x（但此时 B+ 电导还是 1e12）
- 再修正 VSRC_G 后: 62x

**排查过程**:
1. 72x → 498x 的变化是电容符号修正带来的
2. 498x 远超 12AX7 理论最大增益 μ=100，说明还有问题
3. 测量发现"AC peak"值 49.8V 是 DC 偏置 103V 的约一半 → 信号在削波
4. 削波原因是输入信号太大（0.1V），而非模型错误
5. 降低输入到 0.01V 后测得增益 62x
6. 但 62x 的输出偏差从 103V 跳到了 203V → 原来 VSRC_G 从 1e12 降到 1e6 改变了 B+ 的精确度

**真相**: 三个修复叠加，中间状态看起来像是 bug，其实是不同修复在不同阶段生效导致的中间产物。每次修改后都应该重新确认基准。
