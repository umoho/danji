# push_pull 完整推挽调试记录

## 背景

实现完整推挽功率级（12AX7 长尾倒相器 → 耦合电容 → EL84×2 → 推挽变压器 → 8Ω 扬声器），
在 B+ 缓启阶段出现 NaN/Inf，求解器发散。

## 已确认根因

**BE 耦合电感的高效 DC 阻抗 + 耦合电容瞬态 DC 耦合 = 正反馈发散**

1. 耦合电容在 BE 模型中表现为电阻 `Rc = h/C`（0.022µF @ 44.1kHz ≈ 1kΩ）
2. B+ 缓启时 12AX7 屏极 DC (~247V) 通过该电阻直通到 EL84 栅极
3. EL84 在 Vgk=+247V 时 Koren 模型预测电流高达 5A+
4. BE 耦合电感的等效 DC 电阻 `R_eff = L(1-k²)/h ≈ 10.7kΩ`（2.5H, k=0.95）
5. 5A × 10.7kΩ = 53kV 压降 → Vpk 为负 → pentode Vpk 守卫触发 → Ip=0 → Vpk 回到 B+ → 循环

**Newton 迭代在导通/截止间振荡，50 次迭代不收敛。**

## 尝试过的修复

### 1. Snubber 电阻（47Ω 从屏极到 B+）

在电路层面添加 47Ω 电阻与耦合电感初级并联，降低 EL84 看到的 DC 阻抗：
- 总阻抗从 10.7kΩ 降到 ≈ 47Ω
- 5A × 47Ω = 235V 压降，Vpk ≈ B+ - 235，不再为负
- 但因为 CI 状态更新中的 (1-d) 阻尼与并联电阻形成正反馈，
  CI 的 i1_prev 在 20000 步预热中指数增长到 10^99 → NaN

### 2. 矩阵阻尼（在 solver stamp 中直接加 1/R）

在 solver 的 CI stamp 中把阻尼电阻加入 MNA 矩阵，同时 CI 状态更新保持纯 BE 形式。
但 (1-d) 阻尼（state update 中）与 1/R（matrix 中）双重复用导致：
- 有效阻抗计算不一致
- CI 电流源的反馈增益为正 → 指数发散

### 3. DC 直耦替代耦合电容

移除耦合电容，用电阻分压器直接连接 12AX7 屏极和 EL84 栅极：
- 避免 BE 电容瞬态 DC 耦合问题
- 但 12AX7 屏压 ~247V，分压后 EL84 栅压仍为正（Vgk > 0）
- 12AX7 偏置点改变后（Rk 从 47kΩ 改为 1kΩ），屏压降低到 ~50V，
  但 EL84 栅压仍为正 → 静态电流过大

### 4. 提高阻尼系数 d = hR/L

将 CI 状态更新中的阻尼 R 从 150Ω 提高到 10kΩ：
- d = h×10k/2.5 = 0.091，每步衰减 9%
- 但 CI 状态是 state variable，提高 damping 改变物理行为

### 5. 两次预热（先 12AX7 后 EL84）

先让 12AX7 建立工作点（EL84 屏栅接地），待耦合电容充电完毕再引入 EL84。
因 Simulator API 只支持一个 B+ 节点，无法独立控制两路电压。

### 6. VSRC_G 从 1e6 降到 1e4

降低矩阵条件数，改善数值稳定性。对 NaN 无帮助。

### 7. 耦合电感 stamp 改用完整 Z⁻¹ 公式

包含 DCR 的直接 Z⁻¹ 计算，而不是用 (1-d) 近似。
在本次调试中未完全实施，但预计是正确方向。

## 关键教训

1. **BE 耦合电感的高效 DC 阻抗是根本问题**：对于 2.5H 电感，BE 模型
   在 DC 表现为 10.7kΩ（44.1kHz 下）。这个阻抗远高于实际 DCR（~150Ω），
   导致 Newton 迭代严重 overshoot。

2. **耦合电容在 BE 中的 DC 透传**：`Gc = C/h` 使耦合电容在 BE 中
   表现为一个电阻，其阻值与 C 成反比（大 C 似短路，小 C 似高阻）。
   但无论 C 的大小，它无法像在真实电路中那样阻断 DC。

3. **状态变量的正反馈增长**：CI 的 i1_prev 在每次状态更新中可能被当前
   求解的电压放大，形成正反馈。初始 0 值不增长，但任何非零种子
   （如数值噪声、调试用初始值）都会指数发散。

4. **调试策略**：定位 NaN 源头花了大量时间。通过逐步缩小范围，
   最终在 stamp 前打印 I 向量发现 CI 的 i1_prev 已达 10^99。
   但此前数小时的矩阵分析、pivot 检查都无法定位，因为问题不在矩阵
   而在 CI 状态更新的累积效应。

## 下一步方向

1. **双 Simulator 方案**：12AX7 倒相器一个 Simulator，EL84 输出级另
   一个 Simulator。级间通过外部 RC 滤波隔直（如 full_chain.rs 的做法）。
   这是当前 BE 限制下最可靠的方案。

2. **耦合电感模型重写**：把 DCR 纳入 Z⁻¹ 公式的 stamp，同时状态更新
   使用纯 BE 形式（无 (1-d) 阻尼），反馈项改由矩阵自身提供。
   这是根本解决，但需要验证与现有例子的兼容性。

3. **两步预热**：修改 Simulator API 支持多个 B+ 节点，先倒相器预热，
   再 EL84 预热。在 B+ 缓启期间将 EL84 屏栅接地来阻止导通。
